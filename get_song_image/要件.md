# 要件

## ゴール

リストにあるアーティスト名と曲名をもとに、対象楽曲の画像を取得して保存する。

## 前提

- Apple Music API を使用して実現する。
- 入力データとして、少なくとも「アーティスト名」「曲名」を含むリストを扱う。
- 取得対象の画像は、Apple Music API から取得できる楽曲関連のアートワーク画像とする。

## 想定機能

1. アーティスト名と曲名のリストを読み込む。
2. Apple Music API で楽曲検索を行う。
3. 検索結果から対象楽曲を特定する。
4. 楽曲に紐づく画像 URL を取得する。
5. 画像をローカルに保存する。

## 必要な設定

- Apple Music API 利用に必要な認証情報を `.env` に定義する。
- 保存先ディレクトリを `.env` で切り替え可能にする。
- 必要に応じて storefront を指定できるようにする。

## `.env` で管理する項目

- `APPLE_MUSIC_DEVELOPER_TOKEN`
- `APPLE_MUSIC_STOREFRONT`
- `OUTPUT_DIR`

## 実装設計

### 想定ディレクトリ構成

```text
get_song_image/
  .env
  要件.md
  input/
    songs.csv
  output/
    images/
  src/
    main.ts
    config.ts
    types.ts
    readInput.ts
    searchSong.ts
    selectBestMatch.ts
    downloadImage.ts
    utils.ts
```

### 入力仕様

- 入力ファイルは CSV を想定する。
- カラムは最低限 `artist_name`, `song_name` を持つ。
- 任意で `image_filename` を持てるようにし、未指定時はアーティスト名と曲名から自動生成する。

### 出力仕様

- Apple Music API の検索結果から取得した画像をローカルへ保存する。
- 保存先は `OUTPUT_DIR` で指定する。
- ファイル名は以下の優先順位で決定する。
  1. `image_filename`
  2. `artist_name_song_name`
- ファイル名に使用できない文字はサニタイズする。

### 処理フロー

1. `.env` を読み込み、必要な設定値を検証する。
2. 入力 CSV を読み込む。
3. 1行ずつ `artist_name` と `song_name` で Apple Music API の楽曲検索を行う。
4. 検索結果から最も一致度の高い楽曲を選ぶ。
5. 選択した楽曲の artwork 情報から画像 URL を生成する。
6. 画像をダウンロードして `OUTPUT_DIR` に保存する。
7. 成功・失敗の結果をログに出力する。

### Apple Music API 利用方針

- 楽曲検索には Apple Music API の catalog search エンドポイントを使用する。
- リクエスト時に `Authorization: Bearer {APPLE_MUSIC_DEVELOPER_TOKEN}` を付与する。
- storefront は `APPLE_MUSIC_STOREFRONT` の値を使用する。
- 検索キーワードは `アーティスト名 曲名` を結合して送る。

### 楽曲選定ロジック

- 第1条件として曲名の完全一致または高い部分一致を優先する。
- 第2条件としてアーティスト名の一致度を確認する。
- 完全一致がない場合は、先頭候補をそのまま使うのではなく一致スコアで比較する。
- 一致度が一定未満の場合は未取得として扱い、ログに残す。

### 画像 URL の扱い

- Apple Music API の artwork 情報に含まれる URL テンプレートを利用する。
- `{w}` と `{h}` は固定値に置換して保存用 URL を生成する。
- 画像サイズは初期値として 1000x1000 を想定し、必要なら設定化する。

### モジュール設計

- `config.ts`
  `.env` の読み込みと設定値検証を行う。
- `readInput.ts`
  CSV を読み込み、入力データを配列化する。
- `searchSong.ts`
  Apple Music API へ検索リクエストを送る。
- `selectBestMatch.ts`
  検索結果から対象楽曲を選定する。
- `downloadImage.ts`
  画像 URL からファイルを保存する。
- `types.ts`
  入力データ、API レスポンス、処理結果の型を定義する。
- `main.ts`
  全体フローを制御するエントリーポイントとする。

### エラー処理

- `.env` に必須値が不足している場合は起動時に終了する。
- API エラー時は対象行を失敗として記録し、次の行の処理は継続する。
- 画像ダウンロード失敗時も同様に継続し、最終的に失敗一覧を確認できるようにする。
- 入力不備がある行はスキップし、原因をログ出力する。

### ログと結果確認

- 実行時に「成功件数」「失敗件数」を最後に集計表示する。
- 失敗理由は `song not found`, `multiple candidates`, `download failed` などに分類する。
- 必要であれば後続で結果 CSV や JSON を出力できる構成にする。

### 今後の拡張候補

- 入力フォーマットを CSV 以外に JSON にも対応する。
- 並列ダウンロード数を制御して高速化する。
- 既存ファイルがある場合は再取得をスキップする。
- 画像サイズや拡張子を設定ファイル化する。

## 補足

- 保存時のファイル名規則や重複時の扱いも今後の設計対象とする。
