# 音楽MBTI診断ツール 設計書

## 1. システム概要
- 目的: 音楽MBTI診断を36問で実施し、診断タイプの概要を表示した上でInstagramの詳細ページへ誘導する。
- 対象: スマホファースト。PC表示にも自然に対応するレスポンシブUI。
- デプロイ/DB: ローカル環境のみを対象。DBは持たず、質問・結果データは静的JSONとして管理する。

## 2. 技術構成
| レイヤ | 技術 | 備考 |
| --- | --- | --- |
| ビルドツール | Vite + npm | `npm create vite@latest myu-mbti -- --template react-ts`
| フロントエンド | React 18 + TypeScript | SPA構成。React Routerで画面遷移を制御。
| UI | Tailwind CSS またはCSS Modules | 白ベース/余白広めのデザインを容易に再現。
| 状態管理 | React Hooks(useState/useReducer/useContext) | 回答進捗・結果をグローバルで共有。
| テスト | Vitest + @testing-library/react | ロジックと主要コンポーネントの単体テスト。
| 静的データ | `/src/data/questions.ts`, `/src/data/results.ts` | JSON/TSで定義しビルドに含める。

## 4. 画面設計
### 4.1 診断画面(DiagnosePage)
- 構成: ヘッダー(ロゴ/タイトル) → 進捗バー → 質問リスト(36問) → 「結果を見る」ボタン。
- 質問UI: 5段階丸ボタンを横並び。両端(L1/R5)を16px大きくし、中央に向かって縮小して視覚的に好みを表現。
- 振る舞い:
  - ページ表示と同時に質問がレンダーされる。
  - 回答未選択の設問がある場合、結果ボタンは`disabled`。押下時には未回答数をトーストで通知。
  - 質問は1～3問単位のアコーディオンで区切り、モバイルでのスクロール疲労を軽減。

### 4.2 結果画面(ResultPage)
- 表示要素: タイプ名、イメージ画像、タイプ説明(200〜250文字)、CTAボタン「詳細な説明を見る」。
- 共有導線: Instagram URLを`window.open`で新規タブ表示。URLは固定。
- 付随情報: 再診断ボタンで回答をリセットしDiagnosePageへ戻る。

## 5. ルーティング
| パス | 画面 | 備考 |
| --- | --- | --- |
| `/` | DiagnosePage | デフォルト。URL直アクセスで診断開始。
| `/result/:type` | ResultPage | 診断完了で遷移。`:type`はMBTI16タイプID。
| `*` | NotFound | ルート外アクセス時に`/`へリダイレクト。

## 6. データ設計
### 6.1 質問データ(`Question`)
```ts
export type Dimension = 'EI' | 'SN' | 'TF' | 'JP';
export type Question = {
  id: string;        // "Q01" ～ "Q36"
  text: string;      // 質問文
  dimension: Dimension; // MBTI軸。各9問。
  weight: number;    // 1.0を基本に、差別化したい設問のみ1.2などを設定。
};
```
- 設問配分: EI/SN/TF/JP各9問。
- 並び: 軸が偏らないようEI→SN→TF→JPを3回繰り返す配列順。

### 6.2 回答データ
```ts
export type AnswerValue = 1 | 2 | 3 | 4 | 5; // 1=左端,5=右端
export type AnswerMap = Record<string, AnswerValue>;
```
- 左端を軸のマイナス方向(例:EIならE寄り)、右端をプラス方向(I寄り)として扱う。

### 6.3 結果データ(`ResultProfile`)
```ts
export type ResultProfile = {
  type: string;          // "INTJ" など
  label: string;         // タイプの表示名(例:「静かなコンダクター」)
  description: string;   // 200〜250文字の概要
  imagePath: string;     // public/images/intj.png
};
```
- `results.ts`で16タイプを定義。
- Instagram遷移URLは定数`INSTAGRAM_URL`で管理。

## 7. 診断ロジック
1. 回答値(1〜5)を -2〜+2 に正規化: `normalized = value - 3`。
2. 軸ごとに`normalized * weight`を合算。
3. 軸スコア >= 0 で軸後半(EIならI)を採用、 <0で軸前半(E)。
4. 4軸を連結しMBTIタイプ(ID)を得る。
5. 最終タイプで`ResultProfile`を取得し結果画面へ渡す。
- バリデーション: 36問すべて回答済みか、`Object.keys(answers).length === 36`で確認。
- テスト: 正規化・集計の単体テストを`src/hooks/__tests__/useDiagnosis.test.ts`に用意。

## 8. 状態管理
- `DiagnosisProvider`で以下をContext化:
  - `answers: AnswerMap`
  - `selectAnswer(questionId, value)`
  - `isComplete`
  - `computeResult()`
- ResultPageでは`location.state`またはContextを参照。直アクセス時は結果計算不可なので`/`へリダイレクト。

## 9. UI/UX要件反映
- カラーパレット: `#FFFFFF`ベースに、アクセントカラー`#7C4DFF`を回答UIに使用。中間トーンに薄紫(#EDE7F6)。
- タイポグラフィ: ベース16px、質問文18px、結果タイプ24px/700。
- 余白: セクション毎に`padding 24px`、回答ボタン間`gap 12px`。
- レスポンシブ
  - モバイル(<=768px): 1列縦スクロール、固定CTAを画面下に配置。
  - デスクトップ: 質問群を2カラム表示しスクロール量を削減。
- アクセシビリティ: `aria-labelledby`で質問と回答グループを紐付け、ボタンには`aria-pressed`を設定。

## 10. ローカル開発手順
1. `npm create vite@latest myu-mbti -- --template react-ts`
2. `cd myu-mbti && npm install`
3. `npm install -D tailwindcss postcss autoprefixer && npx tailwindcss init -p`
4. `npm run dev`で`http://localhost:5173`を起動。
5. 画像素材を`public/images`へ配置。

## 11. テスト方針
- 単体テスト
  - `useDiagnosis`のスコア計算、未回答ガード。
  - `QuestionGroup`の回答イベント発火。
- スナップショット
  - `ResultCard`のレンダリング。
- E2E(任意)
  - Playwrightで36問回答→結果画面遷移を確認。
