# 音楽MBTI診断ツール 要件定義

## 1. 目的
Myuの認知獲得を目的とする。  
ユーザーが音楽MBTI診断を通じて自分のタイプを知り、その後Instagramへ遷移することで、Myuの存在や世界観、関連コンテンツを知る導線を作る。

## 2. 概要
ユーザーは診断ページを開くと、すぐに質問回答画面から開始する。  
全36問の質問に対して、各設問ごとに1〜5段階で回答する。  
すべての設問に回答後、「結果を見る」ボタンを押すと結果ページへ遷移する。  
結果ページでは、診断タイプ名、イメージ画像、説明文を表示する。  
ページ下部の「詳細な説明を見る」ボタンを押すと、指定のInstagramページへ遷移する。

遷移先Instagram:  
`https://www.instagram.com/myu_music_sns`

## 3. ターゲット
1. 音楽にこだわりのある人  
   - アーティスト中心
   - 音楽の好みを自己表現したい人
2. 一般ユーザー  
   - 診断コンテンツが好きな人
   - SNSでシェア可能なコンテンツに興味がある人

## 4. 想定導線
1. X / Instagram / TikTok で診断の存在を知る
2. 診断サイトへ流入する
3. 36問に回答する
4. 診断結果ページを見る
5. 「詳細な説明を見る」からInstagramへ遷移する
6. Instagram上で各タイプの詳細やおすすめ音楽、Myuの紹介を見る
7. Myuのインストール・認知につなげる

# 機能要件

## 1. 診断開始機能
- ユーザーがページを開いた時点で、即座に質問画面が表示される
- 開始ボタンや説明画面は設けず、最初から回答可能な状態とする

## 2. 質問回答機能
- 設問数は36問
- 各設問に対し、1〜5段階で回答できる
- 回答UIは丸型ボタンを使用する
- 丸型ボタンは横一列に並べる
- 両端のボタンを大きくし、中央に向かうほど小さくするデザインとする
- 各設問は1つのみ回答可能
- 回答未選択の設問がある場合、結果表示はできない

## 3. 結果表示機能
- 全36問への回答完了後、「結果を見る」ボタンを押下できる
- 押下後、結果ページへ遷移する
- 結果ページには以下を表示する
  - 診断名
  - イメージ画像
  - タイプ説明文
  - 詳細な説明を見るボタン

## 4. 外部遷移機能
- 「詳細な説明を見る」ボタン押下時、Instagramページへ遷移する
- 遷移先は固定URLとする  
  `https://www.instagram.com/myu_music_sns`

# 非機能要件

## 1. デザイン要件
- 全体は白ベース
- シンプルでミニマルなUI
- 余白を広めにとり、見やすい設計にする
- 質問文が主役になるレイアウトにする
- 選択UIは画像イメージのような丸デザインを採用する
- 色数は少なめに抑える
- 結果ページも同様に白ベースで統一する

## 2. 操作性
- スマホでの閲覧・操作を優先して設計する
- 1問ごとのタップ領域を十分に確保する
- ユーザーが迷わず直感的に回答できること

## 3. 表示速度
- 初回表示を軽くし、診断開始まで待たせない
- 結果ページ遷移時もスムーズに表示されること

# 画面要件

## 1. 診断画面

### 表示項目
- 質問文
- 5段階の回答ボタン
- 次の設問群
- 結果を見るボタン

### 仕様
- ページを開いたらすぐ診断画面を表示する
- 36問を縦に並べる、または数問ずつ見せる構成は実装時に選択可能だが、UI思想としては「すぐ回答に入れること」を優先する
- 全設問回答済みになるまで「結果を見る」ボタンは非活性、もしくは押下時に未回答を知らせる

## 2. 結果画面

### 表示項目
- 診断タイプ名
- イメージ画像
- タイプ説明文
- 詳細な説明を見るボタン

### 仕様
- タイプに応じて表示内容を切り替える
- 説明文は簡潔な概要にとどめる
- 詳細情報はInstagramで補完する前提とする
